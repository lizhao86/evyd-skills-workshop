# Team TODO 管理

通过读写本地 Markdown 文件 `todo/team-todo.md` 管理团队待办事项。无外部 API 依赖。

## 数据文件

- **主数据：** `todo/team-todo.md` — 所有成员的 TODO（workspace 根目录下）
- **成员映射：** `todo/members.md` — 姓名 ↔ userid ↔ 角色

> 路径相对于 workspace 根目录 `/root/.openclaw/workspace/`。

## 数据格式

每条 TODO 一行：
```
- [ ] 任务内容 | Created:YYYY-MM-DD | Due:YYYY-MM-DD
- [ ] 任务内容 | Created:YYYY-MM-DD
```

- `Created`：必填，YYYY-MM-DD 格式
- `Due`：可选，YYYY-MM-DD 格式
- **无其他字段**（无来源、无优先级、无备注）
- 完成的 TODO **直接从文件中删除**，不保留

## 权限模型

根据 `todo/members.md` 中的角色字段判断权限：

| 操作 | admin | member |
|---|---|---|
| 创建 TODO（给任何人） | ✅ | ❌ |
| 查看 TODO | ✅ 全员 | ✅ 仅自己 |
| 更新/完成 | ✅ 全员 | ✅ 仅自己 |
| 删除 TODO | ✅ | ❌ |
| 查看全员汇总 | ✅ | ❌ |

### 身份识别与自动注册

1. 从会话上下文中获取发送者信息（企微 userid、飞书 userid 等）
2. 在 `todo/members.md` 中匹配姓名或 userid
3. **匹配不到时自动注册：**
   - 问用户你叫什么名字
   - 用户回复后，写入 `todo/members.md`（角色 = member）
   - 同时在 `todo/team-todo.md` 新建该成员的 section
   - 回复注册成功提示

## 四个核心操作

### ① 布置任务

**触发词：** "给XX加个todo"、"给XX布置任务"、"新增待办"

**流程：**
1. 解析：目标成员、任务内容、截止日期（可选）
2. 校验权限：仅 admin 可创建
3. 读取 `todo/team-todo.md`
4. 在目标成员的 section 下追加：`- [ ] 任务内容 | Created:YYYY-MM-DD | Due:YYYY-MM-DD`（Due 可选）
5. 若目标成员 section 不存在，新建 `## 姓名` section
6. 写回文件
7. 回复确认

**回复格式（严格遵循）：**

有 Due 时：
```
✅ 已给{姓名}加了待办：{任务内容} | Due:{YYYY-MM-DD}
```

无 Due 时：
```
✅ 已给{姓名}加了待办：{任务内容}
```

**禁止：**
- 不得省略 ✅ 前缀
- 不得添加额外的问句、建议或装饰文字
- 不得重复展示待办列表

### ② 查看待办

**触发词：** "我的待办"、"今天有什么任务"、"待办列表"

**流程：**
1. 识别当前用户身份
2. 读取 `todo/team-todo.md`
3. member：只展示自己的未完成 TODO
4. admin：展示自己的，并提示可用"汇总"查看全员

**输出格式（严格遵循，不得省略或改写任何部分）：**

```
📋 你的待办（{N}条未完成）：

{编号}. {任务内容} | Due:{YYYY-MM-DD} {动态标记}
{编号}. {任务内容} | No Due

搞完了跟我说，我帮你删。
```

**逐行规则：**
1. 第一行固定 `📋 你的待办（{N}条未完成）：`，N = 实际条目数
2. 空一行
3. 每条一行：`{编号}. {任务内容} | Due:{YYYY-MM-DD}` 或 `{编号}. {任务内容} | No Due`
4. 有 Due 的条目在行末追加动态标记（见下表），无 Due 不追加
5. 排序：Due 从近到远，无 Due 放最后
6. 空一行
7. 固定尾句 `搞完了跟我说，我帮你删。`

**动态标记（根据 Due vs 今天日期计算，不可手动编造）：**

| 条件 | 标记 |
|---|---|
| Due < 今天 | 🔥（逾期） |
| Due = 今天 | 🔴（今天截止） |
| Due = 明天 | ⚠️（明天截止） |
| Due > 明天 | 无标记 |
| 无 Due | 无标记 |

**示例（假设今天是 2026-04-13）：**
```
📋 你的待办（4条未完成）：

1. 完成API文档 | Due:2026-04-12 🔥
2. 去NUHS | Due:2026-04-13 🔴
3. 晚上去老巴刹 | Due:2026-04-14 ⚠️
4. 更新 openclaw 版本 | No Due

搞完了跟我说，我帮你删。
```

**禁止：**
- 不得省略 📋 前缀
- 不得改写尾句
- 不得添加「高优/中优/低优」等数据中不存在的字段
- 不得显示 Created 日期（个人查看不需要）
- 不得添加额外的问句、建议或装饰文字

### ③ 自然语言更新

**触发词：** "XX做完了"、"PRD写好了"、"把截止日改到下周五"

**流程：**
1. 识别当前用户身份
2. 读取 `todo/team-todo.md`
3. 在该用户（member）或全员（admin）的 TODO 中模糊匹配最相关条目
4. 若唯一匹配：直接执行
5. 若多条匹配：列出让用户确认
6. 支持的操作：
   - **标记完成：** 直接从文件中删除该行
   - **改截止日：** 更新 Due 字段（YYYY-MM-DD 格式）
7. 写回文件
8. 回复确认

**回复格式（严格遵循）：**

标记完成：
```
✅ 已删除：{任务内容}
```

改截止日：
```
✅ 已更新：{任务内容} | Due:{YYYY-MM-DD} → {YYYY-MM-DD}
```

多条匹配时列出确认：
```
❓ 匹配到{N}条，你说的是哪个？

1. {任务内容A} | Due:{YYYY-MM-DD}
2. {任务内容B} | No Due

回复编号即可。
```

**禁止：**
- 不得省略 ✅/❓ 前缀
- 不得添加额外的问句、建议或装饰文字

### ④ 全员汇总

**触发词：** "汇总"、"全员进度"、"团队报告"、"今日报告"

**权限：** 仅 admin

**流程：**
1. 读取 `todo/team-todo.md`
2. 按成员分组统计：未完成数、逾期数
3. **严格按** [references/report-format.md](references/report-format.md) **的模板输出，不得省略或改写任何部分**

## 注意事项

- 每次写文件前先读取最新内容，避免覆盖并发修改
- 完成的 TODO 直接删除，不做标记保留
- 不依赖任何外部 API（企微待办、智能表格等均不使用）
