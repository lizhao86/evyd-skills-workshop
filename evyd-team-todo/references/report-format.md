# 汇总报告格式（严格遵循，不得省略或改写任何部分）

## 模板

```
📊 团队待办汇总 {YYYY-MM-DD}

{姓名}：{N}条未完成（{M}项逾期）
 - [ ] {任务内容} | Due:{YYYY-MM-DD} {优先级标记}
 - [~] {任务内容} | Due:{YYYY-MM-DD} {优先级标记}
 - [ ] {任务内容} | No Due Date

{姓名}：{N}条未完成（{M}项逾期）
 - [ ] {任务内容} | Due:{YYYY-MM-DD} {优先级标记}

─────────────────
全员统计：{X}条未完成 / {Y}项逾期
```

## 逐行规则

1. 第一行固定 `📊 团队待办汇总 {YYYY-MM-DD}`，日期 = 当天
2. 空一行
3. 每个成员一个 block：
   - 标题行：`{姓名}：{N}条未完成（{M}项逾期）`，N = `[ ]` + `[~]` 条目数（不含 `[x]`），M=0 时写 `（0项逾期）`
   - 每条 TODO 一行：` - [ ] {任务内容} | Due:{YYYY-MM-DD} {标记}` 或 ` - [~] {任务内容} | ...` 或 ` - [ ] {任务内容} | No Due Date`
   - `[~]` 进行中的条目保留 `[~]` 前缀；`[ ]` 未开始的保留 `[ ]` 前缀
   - 缩进子项跟随父条目展示（如有）
4. 成员之间空一行
5. 分隔线 `─────────────────`
6. 统计行 `全员统计：{X}条未完成 / {Y}项逾期`
7. 如有 `[x]` 已完成条目（保留记录模式），在统计行后另起：`─── 已完成（{K}条）───` 然后逐行列出，无编号

## 排序规则

- 成员之间：按逾期数从多到少排序（逾期多的排前面）
- 每个成员的待办：按 Due 从近到远，无 Due 放最后

## 优先级标记（根据 Due vs 今天日期计算，不可手动编造）

| 条件 | 标记 | 含义 |
|---|---|---|
| Due < 今天 | P0 | 逾期 |
| Due = 今天 | P1 | 今天截止 |
| Due = 明天 | P2 | 明天截止 |
| Due > 明天 | 无标记 | |
| 无 Due | 无标记 | |

**不使用 emoji（🔥🔴⚠️等），统一用 P0/P1/P2 纯文本标记。**

## 禁止

- 不得添加「高优/中优/低优」等数据中不存在的字段
- 不得省略 📊 前缀、分隔线、统计行
- 不得添加额外的问句、建议或装饰文字
- 不得用 emoji（🔥🔴⚠️等）替代 P0/P1/P2 标记
- 无 Due 的条目必须写 `No Due Date`，不得写 `No Due` 或留空
