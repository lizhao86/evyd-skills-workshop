# 汇总报告格式（严格遵循，不得省略或改写任何部分）

## 模板

```
📊 团队待办汇总 {YYYY-MM-DD}

{姓名}：{N}条未完成（{M}项逾期）
 - [ ] {任务内容} | Due:{YYYY-MM-DD} {动态标记}
 - [ ] {任务内容} | No Due

{姓名}：{N}条未完成（{M}项逾期）
 - [ ] {任务内容} | Due:{YYYY-MM-DD} {动态标记}

─────────────────
全员统计：{X}条未完成 / {Y}项逾期
```

## 逐行规则

1. 第一行固定 `📊 团队待办汇总 {YYYY-MM-DD}`，日期 = 当天
2. 空一行
3. 每个成员一个 block：
   - 标题行：`{姓名}：{N}条未完成（{M}项逾期）`，M=0 时写 `（0项逾期）`
   - 每条 TODO 一行：` - [ ] {任务内容} | Due:{YYYY-MM-DD} {标记}` 或 ` - [ ] {任务内容} | No Due`
4. 成员之间空一行
5. 分隔线 `─────────────────`
6. 统计行 `全员统计：{X}条未完成 / {Y}项逾期`

## 排序规则

- 成员之间：按逾期数从多到少排序（逾期多的排前面）
- 每个成员的待办：按 Due 从近到远，无 Due 放最后

## 动态标记（根据 Due vs 今天日期计算，不可手动编造）

| 条件 | 标记 |
|---|---|
| Due < 今天 | 🔥（逾期） |
| Due = 今天 | 🔴（今天截止） |
| Due = 明天 | ⚠️（明天截止） |
| Due > 明天 | 无标记 |
| 无 Due | 无标记 |

## 示例（假设今天是 2026-04-13）

```
📊 团队待办汇总 2026-04-13

李四：2条未完成（1项逾期）
 - [ ] 部署测试环境 | Due:2026-04-12 🔥
 - [ ] 准备周报 | No Due

赵立：3条未完成（0项逾期）
 - [ ] 去NUHS | Due:2026-04-13 🔴
 - [ ] 晚上去老巴刹 | Due:2026-04-14 ⚠️
 - [ ] 更新 openclaw 版本 | No Due

─────────────────
全员统计：5条未完成 / 1项逾期
```

## 禁止

- 不得出现 `[x]` 已完成项（完成即删除，不存在于数据中）
- 不得出现「已完成」计数
- 不得添加「高优/中优/低优」等数据中不存在的字段
- 不得省略 📊 前缀、分隔线、统计行
- 不得添加额外的问句、建议或装饰文字
