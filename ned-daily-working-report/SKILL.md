# Ned 每日工作报告生成器

## 资源文件

- **模板**：`assets/template.md` - 日报模板和格式规范
- **脚本**：`scripts/generate_daily_report.py` - 自动化生成脚本

## 工作流程

### 1. 读取当天会议纪要

从以下路径读取当天的会议纪要文件：
- 路径：`/Users/Li.ZHAO/我赵立的仓库/21-Work-会议纪要 💼`
- 筛选规则：文件名包含当天日期（格式：`💼 2026-03-10 XXX.md`）

### 2. 读取当天个人日记

从以下路径读取当天的个人日记文件：
- 路径：`/Users/Li.ZHAO/我赵立的仓库/10-Notes-晨夕日记 📝/YYYY/MM/`
- 筛选规则：文件名包含当天日期（格式：`📝 2026-03-10 XXX.md`）
- 注意：跳过"4 每日 Todo 一览"部分的内容

### 3. 生成日报内容

#### 格式要求

```markdown
## 一、工作流水账

### 1. [会议/工作名称]
**核心内容**：简要归纳核心内容（1-3 句话）。
**来源**：[[文件名]]

### 2. [会议/工作名称]
**核心内容**：简要归纳核心内容。
**来源**：[[文件名]]

## 二、下一步汇总

- [待办事项 1]
- [待办事项 2]
- [待办事项 3]

## 三、来源

- [[会议纪要文件名 1]]
- [[会议纪要文件名 2]]
- [[个人日记文件名 1]]
```

#### 核心规则

1. **文件名引用格式**：使用 Obsidian 链接格式 `[[文件名]]`（不带 .md 后缀）
2. **文件名空格**：日期后面有一个空格（如 `💼 2026-03-10 「SengKang」...`）
3. **简洁优先**：每个会议只写核心内容，不展开细节
4. **来源标注**：每个会议/工作都要标注来源
5. **无标题**：不需要 `# 📅 日报 2026-03-10` 这样的标题
6. **无分隔线**：不需要 `---` 分隔线
7. **来源汇总**：在「## 三、来源」部分列出所有参考过的文件，使用 Obsidian 链接格式

### 4. 保存文件

- 保存路径：`/Users/Li.ZHAO/我赵立的仓库/24-Work-周报汇总/`
- 文件名格式：`YYYY-MM-DD EVYD 日报.md`（如 `2026-03-10 EVYD 日报.md`）
- 注意：日期和"EVYD"之间有一个空格

### 5. 汇报规则

1. **有内容**：告诉老板"日报已生成！✅ 文件位置：[路径]"
2. **没内容**：告诉老板"今天没有工作记录 📋"
3. **有内容但不相关工作**：告诉老板"今天没有工作记录 📋"（如纯个人感悟、无工作内容）
