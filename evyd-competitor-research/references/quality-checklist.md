# 交付前质检清单

> 报告合并完成后、输出到目标渠道前，逐条检查。任何一项不通过必须修正后再交付。

## 内容质量（7 项）

- [ ] **C1 叙事性**：报告读起来像分析文章而非数据堆砌？每节有判断性开头句？
- [ ] **C2 证据密度**：关键判断是否都有来源支撑？`[待验证]` 标记不超过 3 处？
- [ ] **C3 禁用词检查**：全文无 `writing-rules.md` 禁用词表（中文 + 英文）中的任何词汇？
- [ ] **C3b 禁用句式检查**：全文无 `writing-rules.md` 禁用句式中的 6 种结构性 AI 腔模式？
- [ ] **C4 具体性**：数字、客户名、产品名是否具体？无"广泛""众多""大量"等空洞修饰？
- [ ] **C5 EVYD 视角**：EVYD Comparison 和 SWOT 节是否包含可操作建议（而非泛泛对比）？
- [ ] **C6 诚实性**：信息缺口是否明确标注而非用模糊语言绕过？Research Log 是否记录了搜索盲区？
- [ ] **C7 交叉洞察**：Cross-Axis Insights 节是否从时间+竞争两维度交叉分析，而非简单重复前文？

## 结构完整性（4 项）

- [ ] **S1 模板合规**：report-template.md 中所有必填节均已填写？
- [ ] **S2 链接完整**：Links 节包含所有正文引用的 URL，且已去重？
- [ ] **S3 分类命名**：文件名符合 taxonomy.md 的 System + Module 命名规则？
- [ ] **S4 格式规范**：无 Markdown 表格（飞书不渲染）？列表和段落格式正确？

## 可操作性（3 项）

- [ ] **A1 威胁等级**：Normalized Summary 中的 threat level 有依据支撑？
- [ ] **A2 战术建议**：EVYD tactics 具体到团队（销售/交付/产品）可直接行动？
- [ ] **A3 后续方向**：Similar Competitor Recommendations 提供了 3 家相关公司及选择理由？

## HITL 就绪（2 项）

- [ ] **H1 BOSS 节生成**："PLEASE READ BEFORE SHOW TO BOSS" 节已生成，截图清单和汇报框架中的 `{Competitor}` 已替换为实际公司名？
- [ ] **H2 语气自查**：随机抽查 3 段正文，对照 `writing-rules.md` 的好/坏对比示例，确认语气接近"人话"而非"AI 腔"？
