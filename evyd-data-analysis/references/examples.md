> 所有案例展示完整的来源标注和 Action 细化格式。

## 案例 1：App DAU 环比下降

用户请求：
`3 月 DAU 比 2 月降了 15%，帮我分析原因。`

---

### Data Facts

- [已知] 3 月 DAU 均值 10,490，2 月均值 12,340，环比 -15.0%（p=0.008，显著）
  [来源: daily_metrics.xlsx / Sheet "DAU" / dau 列，avg(3月) vs avg(2月)，t-test p=0.008]
- [已知] 下降拐点在 3/15，前 14 天均值 12,100，后 16 天均值 9,200
  [来源: 同上，按日期切分 3/1-3/14 vs 3/15-3/31 计算均值]
- [已知] 微信渠道 DAU 降幅 22%，贡献总跌幅的 78%
  [来源: daily_metrics.xlsx / Sheet "DAU" / channel="wechat"，delta_wechat / delta_total = 78%]
- [已知] App Store 渠道 DAU 降幅 3%，不显著（p=0.41）
  [来源: 同上 / channel="appstore"]
- [已知] 新用户日均新增从 820 降至 610（-25.6%），老用户回访基本持平
  [来源: daily_metrics.xlsx / Sheet "new_users" / new_user_cnt 列]

### Insights

1. [已知] 微信渠道是主因，贡献 78% 跌幅
   [来源: 同 Facts 第 3 条的渠道拆解]
2. [推断] 3/15 微信入口改版可能是触发点
   [依据: 拐点日期与改版日期吻合，但无 AB 数据直接证明因果]
3. [已知] 老用户活跃度未受影响，问题集中在新用户获取
   [来源: Facts 第 5 条，老用户 DAU 环比变化 < 2%（不显著）]
4. [假设] 若微信渠道恢复至 2 月水平，DAU 可恢复至约 11,800（距 2 月仍差 4.4%）
   [计算: 10490 + (12340-10490) × 78% × (22%/15%) ≈ 11800]

### Actions

**1. 排查微信入口改版影响**
- 负责人：产品/增长
- 分析方法：从埋点平台导出 3/1-3/31 微信入口页 Page UV（按日），以 3/15 为分割线对比前后两段均值；同时导出 App Store 渠道同时段数据作为对照组
- 判定标准：微信入口页 UV 在 3/15 后下降 >10% 且 App Store 无同等跌幅 → 确认改版影响
- 观察期：7 天（拿到数据后）
- 风险：若 3/15 同时有推送/投放策略调整需逐一排除

**2. 补充 App Store 投放弥补新增缺口**
- 负责人：投放
- 分析方法：对比当前 App Store 日新增（610）与目标值（820），计算缺口 210/天；参考历史 CPI 估算所需追加预算
- 判定标准：连续 7 天日新增 > 800 → 缺口已补；CPI 上升 > 20% → 需调整素材或渠道
- 观察期：14 天
- 风险：Q1 投放竞争加剧，CPI 可能高于历史水平

**3. （暂不行动）老用户活跃稳定，继续监控**
- 负责人：BI
- 分析方法：每周导出老用户 DAU 及 D7 留存率，按周汇总趋势
- 判定标准：连续 3 周留存下降 > 1pp → 升级为正式分析课题
- 观察期：持续

### PPT 联动说明

生成 content.json 时的关键规则（以本案例为例）：

**freeform 数字卡片页（替代 stat_highlight）：**
- 每个核心数字用一个 rect 卡片（fill white/card, radius 0.2）+ 内部 text 组合
- 大数字 sz ≥ 44（bold, color accent），标签 sz 20，说明 sz 18（color text_dim）
- 2-4 个卡片横排，间距均匀

**来源标注（每个数据页必须有）：**
- 独立 text element，不嵌入任何卡片或正文段落
- 位置：y ≥ 9.5（页面底部），x = 1.0, w = 18
- 格式：`来源: {filename} / Sheet "{name}" / {column 或 calculation}`
- sz 12, color text_dim

**错误示范（不要这样做）：**
```json
{ "kind": "text", "text": "DAU 环比下降 15%\n来源: daily_metrics.xlsx", "sz": 18 }
```
来源嵌在正文里，审阅者无法快速定位。

**正确做法：**
```json
{ "kind": "text", "x": 1.5, "y": 3.0, "text": "DAU 环比下降 15%", "sz": 18 },
{ "kind": "text", "x": 1.0, "y": 10.0, "w": 18, "h": 0.5, "text": "来源: daily_metrics.xlsx / Sheet \"DAU\" / dau 列 avg(3月) vs avg(2月)", "sz": 12, "color": "text_dim" }
```

## 案例 2：问诊转化漏斗诊断

用户请求：
`问诊下单量本月降了，帮我看看漏斗。`

---

### Data Facts

- [已知] 问诊漏斗（本月 vs 上月）：

```
首页访问    280,000 → 265,000  (-5.4%)
科室搜索    142,000 → 128,000  (-9.9%)
医生列表     98,000 →  84,000  (-14.3%)  ← 最大流失环节
医生详情     67,000 →  58,000  (-13.4%)
发起问诊     23,000 →  18,500  (-19.6%)
支付完成     19,500 →  15,200  (-22.1%)
```

- [已知] "科室搜索 → 医生列表"转化率从 69.0% 降至 65.6%（-3.4pp，p=0.02，显著）
- [已知] "医生详情 → 发起问诊"转化率从 34.3% 降至 31.9%（-2.4pp，p=0.08，边缘显著）
- [已知] 支付环节转化率 84.8% → 82.2%（-2.6pp，p=0.15，不显著）

### Insights

1. [已知] 最大损失在"科室搜索→医生列表"，绝对流失量增加 7,400 人
2. [推断] 医生列表页的排序/筛选体验可能影响了用户决策（需查看页面改动记录）
3. [已知] 从"发起问诊"到"支付"的转化基本稳定，下游不是问题
4. [假设] 若"科室搜索→医生列表"转化率恢复至上月水平，预计可多挽回约 4,500 个问诊 (+24%)

### Actions

1. **排查医生列表页近期改动** — 产品 — 验证："搜索→列表"转化率恢复至 69% — 7 天 — 风险：若无改动需看内容质量
2. **分析"发起问诊"流失用户特征** — BI — 验证：定位高流失客群 — 5 天 — 风险：埋点覆盖可能不全

## 案例 3：功能渗透率分析

用户请求：
`帮我看一下各功能的渗透率，哪些功能该重点推。`

---

### Data Facts

- [已知] 本月 MAU 85,000，各功能渗透率：

```
在线问诊      42.3%   (35,955 人)
购药          38.7%   (32,895 人)
健康资讯      28.1%   (23,885 人)
体检预约       8.4%   (7,140 人)
慢病管理       6.2%   (5,270 人)
AI 症状自查   12.7%   (10,795 人)
```

- [已知] AI 症状自查用户中 61% 会继续发起问诊（交叉渗透率最高）
- [已知] 慢病管理用户的 D30 留存 52%，远高于总体 D30 留存 8%
- [已知] 体检预约和慢病管理的渗透率均 < 10%，且环比无增长

### Insights

1. [已知] AI 症状自查是问诊的高效前置入口（61% 转化至问诊）
2. [已知] 慢病管理虽渗透率低但留存贡献极大（D30 留存 52% vs 总体 8%）
3. [推断] 慢病管理用户为高价值"锚定用户"，提升渗透率可系统性改善留存
4. [假设] 若慢病管理渗透率从 6.2% 提升至 12%，总体 D30 留存可提升约 2pp

### Actions

1. **在问诊结束后推荐慢病管理入口** — 产品 — 验证：慢病管理渗透率 — 30 天 — 风险：非慢病用户误导
2. **强化 AI 症状自查 → 问诊的引导链路** — 产品/运营 — 验证：自查→问诊转化率 — 14 天 — 风险：体验侵入感
3. **体检预约暂不投入增长资源** — 决策依据：渗透低 + 留存贡献不明显 — 后续如有新数据再评估

## 案例 4：活动复盘（促销 ROI）

用户请求：
`上周做了问诊 1 元活动，帮我复盘值不值得继续做。`

---

### Data Facts

- [已知] 活动期（7 天）：问诊订单 8,200 单，活动前同期 3,100 单（+164.5%）
- [已知] 活动补贴总额 ¥41,000（平均每单补贴 ¥5.0）
- [已知] 活动期新用户占比 62%，常规期新用户占比 28%
- [已知] 活动新用户 D7 留存 11%，常规新用户 D7 留存 22%
- [已知] 活动结束后第一周问诊量 3,400 单（回到基线水平，无明显提升）

### Insights

1. [已知] 活动带来 5,100 增量订单（+164%），但活动后无残留效应
2. [已知] 活动拉新用户质量显著低于自然新用户（D7 留存 11% vs 22%，p<0.01）
3. [推断] 大部分增量为价格敏感用户，非目标用户群
4. [已知] 增量收入 ¥8,200（按活动后常规客单价 ¥45 × 增量中预期复购率估算），补贴 ¥41,000 → ROI 为负

### Actions

1. **不建议按原方案继续做 1 元活动** — 决策层 — 依据：ROI 为负，且拉新质量差
2. **下次活动改为"老用户回流"方向** — 运营/增长 — 验证：回流用户 D7 留存 > 15% — 验证周期 14 天 — 风险：老用户池有限
3. **设定活动准入门槛**（如注册 > 7 天）— 产品 — 验证：活动用户 D7 留存 — 下次活动时 — 风险：转化率可能下降
