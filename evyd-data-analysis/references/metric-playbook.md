# 常见指标与拆解手册

## 增长类

### 用户规模

- `DAU` = 当日打开 App 的独立用户数
- `WAU` = 7 天内打开 App 的独立用户数
- `MAU` = 30 天内打开 App 的独立用户数
- `DAU/MAU` = 用户粘性，健康值 > 20%
- `新增用户` = 当日首次注册 / 首次激活（需明确口径）

常见拆法：
- 渠道（App Store / 微信 / 直接访问 / 推送 / 外部投放）
- 地区 / 城市
- 客群（新用户 / 老用户 / 回流用户）
- 设备（iOS / Android / H5）
- 自然周 / 自然月 / 活动期

### 功能渗透率

- `功能渗透率` = 使用某功能的用户数 / DAU × 100%
- `功能频次` = 某功能总使用次数 / 使用该功能的用户数
- `核心功能 DAU` = 使用核心功能（如问诊、购药）的日活

拆法：新老用户 × 渠道 × 入口路径

### 增长率

- `环比增长率` = (本期 - 上期) / 上期 × 100%
- `同比增长率` = (本期 - 去年同期) / 去年同期 × 100%
- `CAGR` = (终值/初值)^(1/年数) - 1

注意：
- 基数小时增长率失真，需同时报告绝对值
- 节假日、活动期需标注，避免误判趋势

## 漏斗类

### 通用漏斗

```
访问 → 注册 → 激活 → 核心功能使用 → 付费
```

### 问诊漏斗（医疗 App 典型）

```
首页访问 → 科室/症状搜索 → 医生列表浏览 → 医生详情页 → 发起问诊 → 支付 → 完成问诊 → 评价
```

### 购药漏斗

```
首页/搜索 → 药品详情 → 加入购物车 → 提交订单 → 支付 → 确认收货
```

### 漏斗输出要求

- 每层转化率（= 本层人数 / 上层人数）
- 每层绝对流失量（= 上层人数 - 本层人数）
- 最大损失环节（绝对流失量最大的那一步）
- 分群差异：按新老用户 / 渠道 / 地区 / 设备拆

### 漏斗分析常见误区

- 只看总转化率，不看每步 → 无法定位问题
- 时间窗不一致 → 同一个用户被重复计入
- 不区分新老用户 → 新用户低转化拉低总体，误判为产品问题

## 留存类

### 指标定义

- `次日留存 (D1)` = Day1 回访用户 / Day0 新增用户
- `7日留存 (D7)` = Day7 回访用户 / Day0 新增用户
- `30日留存 (D30)` = Day30 回访用户 / Day0 新增用户
- `周留存` = 第N周有活跃的用户 / 注册周用户
- `复购率` = 2次及以上购买用户 / 有购买用户

### 健康基准（医疗互联网 App 参考）

| 指标 | 良好 | 一般 | 需关注 |
|---|---|---|---|
| D1 留存 | > 35% | 20-35% | < 20% |
| D7 留存 | > 15% | 8-15% | < 8% |
| D30 留存 | > 8% | 4-8% | < 4% |
| DAU/MAU | > 20% | 10-20% | < 10% |

注意：以上为行业参考，具体因 App 类型差异大。

### Cohort 分析要点

- 按注册月/首购月/首诊月分组
- 横轴：注册后第 N 周/月
- 纵轴：不同批次（cohort）
- 值：留存率

关键看：
- 不同 cohort 的留存曲线是否一致 → 一致说明产品稳定，差异大说明获客质量或产品体验变了
- 留存变差发生在第几周 → 定位流失节点
- 特定 cohort 异常 → 关联该批次的获客渠道、产品版本、活动

### 留存分析常见误区

- 只看总体留存，不看 cohort → 获客结构变化会影响总体留存
- 把"获客质量变差"误判为"产品体验变差" → 需拆开新老用户
- 样本量太小的 cohort 直接下结论 → 标记 `[样本不足]`

## 收入类

### 指标定义

- `GMV` = 总交易额（含未支付/退款前）
- `实际收入` = GMV - 退款 - 优惠券
- `ARPU` = 收入 / 活跃用户数
- `ARPPU` = 收入 / 付费用户数
- `付费率` = 付费用户数 / 活跃用户数
- `客单价` = 收入 / 订单数

### 收入拆解公式

```
收入 = DAU × 付费率 × 客单价
     = 流量 × 转化率 × ARPU
```

量价拆解：
```
收入变化 = 量变化贡献 + 价变化贡献 + 交叉项
        ≈ ΔQ × P₀ + Q₀ × ΔP
```

常见拆法：
- 量 × 价
- 品类/服务结构（问诊 / 购药 / 健康管理 / 会员）
- 渠道结构
- 新老用户
- 折扣与补贴影响

## 效率类

### 指标定义

- `问诊响应时长` = 医生首次回复时间 - 用户发起问诊时间
- `问诊完成时长` = 问诊结束时间 - 发起时间
- `配送时效` = 用户收货时间 - 下单时间
- `客服首响时长` = 客服首次回复 - 用户发起咨询
- `客服解决率` = 一次解决的工单 / 总工单

### 效率拆法

- 单位产出
- 单位成本
- 周期时长
- 资源利用率
- 异常损耗

## 建议动作模板

分析完成后，Action 格式：

```
动作 + 负责人类型 + 验证指标 + 观察周期 + 风险
```

示例：
- 优化问诊入口页文案 — 产品/增长 — 验证：问诊发起转化率 — 7 天 — 风险：样本量可能不足
- 调整低 ROI 渠道预算 — 投放 — 验证：获客成本和 D7 留存 — 14 天 — 风险：季节性扰动
- 针对流失用户发召回 Push — 运营 — 验证：7 天回流率 — 7 天 — 风险：Push 疲劳
- 暂不行动，继续监控 — BI — 验证：周度报表 — 持续 — 说明：变化不显著
